# Deployment

Each app remains independently deployable:

- `apps/hypermyths` -> `hypermyths.com`
- `apps/hypertian` -> `hypertian.com`
- `apps/cancerhawk` -> `cancerhawk.org`
- `apps/hyperkaon` -> `hyperkaon.com`
- `apps/polymyths` -> `polymyths.com`

Future HyperMyths hub paths can cross-link the focused doors:

- `hypermyths.com/ecosystem`
- `hypermyths.com/attention`
- `hypermyths.com/research`
- `hypermyths.com/compute`
- `hypermyths.com/intelligence`

Use one Vercel project per app or equivalent isolated deployments. Do not assume one production deploy for the whole monorepo.
