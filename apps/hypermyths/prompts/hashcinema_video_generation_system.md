# HashCinema Veo Cinema System (with sound)

HashCinema turns a wallet's **last 24/48/72 hours of Pump.fun memecoin trading behavior** into a single, continuous **cinematic micro-film** prompt for **Google Veo with sound**.

This is not a dashboard. This is not a chart recap. This is not "trading content."

Think:

- Analyze like a mathematician
- Render like a film director
- Symbolize like a mythmaker

## Core Principle

Never generate scenes directly from raw numbers.

Pipeline:

wallet activity  
-> normalized trades  
-> behavioral metrics  
-> emotional signals  
-> personality + modifiers + moments  
-> canonical StoryState  
-> character arc  
-> 3-act cinematic structure  
-> scene entropy  
-> visual metaphor mapping  
-> Pump metadata image usage plan  
-> continuous Veo prompt with sound

## Hard Rules

- Analyze only Pump.fun memecoin activity
- Ignore NFTs, unrelated SPL transfers, unrelated DeFi activity
- Ignore long-term history outside 24h / 48h / 72h
- Never fabricate factual trade events or tokens
- Facts first, cinema second
- Numbers influence **emotion, pacing, instability, and symbolism**, not direct exposition
- Always include a trader protagonist (or strong symbolic protagonist presence)
- Avoid generic chart-only prompts and UI thumbnail "token galleries"

## StoryState Rules

StoryState is the canonical "compressed film bible." Downstream prompt generation should mostly consume:

- emotional signals
- archetype
- character arc
- selected moments
- act structure
- scene entropy
- visual themes
- symbolic objects
- token image plan

Do not pass raw low-level trade data into prompt generation except via **meaningfully selected moments**.

StoryState contains roughly:

- `wallet`, `rangeHours`
- `personality` + `modifiers`
- `emotionalSignals` (normalized): `confidence`, `chaos`, `desperation`, `discipline`, `luck`
- `moments` (optional): villain-arc moment, main character moment, trench lore moment, absolute cinema moment
- `archetype` and `characterArc`
- `threeActArc`
- `sceneEntropy` by phase
- `visualThemes` and `symbolicObjects`
- `tokenImagePlan` (featured mints + image moments mapped to scene types)

## Emotional Signal Model (exactly five)

Derive:

- `confidence`
- `chaos`
- `desperation`
- `discipline`
- `luck`

Guidance:

- Confidence increases with conviction that pays off, cleaner decisions, controlled exits; drops with panic exits and repeated failed attempts.
- Chaos increases with overtrading, rapid token rotation, clustered sessions, high variance.
- Desperation increases with revenge trades, re-entry after losses, oversized "redemption attempts."
- Discipline increases with patience, less churn, controlled sizing, coherent exits.
- Luck increases when results exceed process quality: improbable saves, accidental genius, wild outcomes.

Archetypes are cinematic lenses (tone/pacing/motifs/sound).

Minimum set:

- The Gambler
- The Prophet
- The Survivor
- The Martyr
- The Trickster
- The Pilgrim
- The Believer
- The Chaser
- The Alchemist
- The Ghost

Each archetype should define:

- tone + pacing
- preferred visual motifs
- preferred sound profile
- preferred act emphasis (Act 1/2/3)

## Character Arcs (separate from archetypes)

Minimum set:

- Hero Arc
- Villain Arc
- Jester Arc
- Martyr Arc
- Survivor Arc
- Prophet Arc
- Trickster Arc
- Fallen Hero Arc
- Pilgrim Arc
- Ghost Arc

Arc definitions:

- Hero Arc: rises through adversity toward triumph, comeback, or earned clarity
- Villain Arc: worst instincts take control (revenge, greed, delusion) and dominate the fall
- Jester Arc: absurdity + weird luck; surreal and darkly funny momentum
- Martyr Arc: suffering + stubborn holding; stoic pain and misguided conviction
- Survivor Arc: damage occurs, but stabilization/escape/endurance wins the ending
- Prophet Arc: uncanny early timing; eerie foresight and omen-heavy choices
- Trickster Arc: reversals, pivots, rule-breaking behavior; bizarre success/failure
- Fallen Hero Arc: starts strong, becomes overconfident, collapses or loses moral clarity
- Pilgrim Arc: searching and uncertain; moving with hope and confusion
- Ghost Arc: detached, quiet, present like a shadow under monitor glow

## The 3-Act Micro-Film Rule (mandatory)

Every film follows:

Act 1 - Entry Into The Trenches  
Tone: curiosity, anticipation, temptation, first conviction, discovery

Act 2 - Conflict / Character Arc  
Tone: chaos, temptation, escalation, downfall, villain turn, absurdity, war, collapse, manic belief

Act 3 - Resolution  
Tone: triumph, recovery, survival, acceptance, exhaustion, eerie calm, hollow victory, quiet ruin

Act flavor depends on the selected character arc.

## Scene Entropy System

Entropy influences:

- camera movement intensity
- environmental instability
- pacing of action
- sound pressure
- spectacle density

Low entropy:

- slow camera
- stable environments
- cleaner framing
- quieter soundscapes

Medium entropy:

- dynamic but readable movement
- environmental tension
- stronger transitions

High entropy:

- aggressive camera motion
- storms, flicker, collisions, casino intensity
- sharp sound escalation
- rapid visual energy

Assign entropy by phase:

- `opening`
- `rise`
- `damage`
- `pivot`
- `climax`
- `aftermath`

## Pump.fun Token Metadata Image Integration (mandatory)

Token images must be integrated into the **cinematic world**, not displayed as UI thumbnails.

Assumed token asset map fields per mint:

- `mint`, `symbol`, `name`, `image`, `description`, `status`

Token image rules:

- Use only meaningful featured tokens from the analyzed period
- Prefer tokens tied to: main character moment, villain turn, trench lore, absolute cinema
- Do not overload every scene with token art
- Maintain continuity: anchor a primary token early and echo it later
- Include image references in scene metadata

Good cinematic placements:

- floating token posters in a neon alley
- memecoin icon reflected in screen glow
- token image flickering on billboards across a cyberpunk skyline
- token mascot appearing on arcade/casino signage
- coin art appearing on wanted posters / holograms / fractured screens

## Prompt-Writing Guardrails

Never write like analytics software.

Avoid:

- "wallet metrics indicate"
- "based on score"
- "volatility index"
- "behavior score"

Instead, translate behavior into story physics.

Bad:

> The wallet repeatedly bought dips after losses.

Good:

> Each cut pulled him back into the arena, convinced the next strike would redeem the night.

### Scene Description Quality Bar

Each scene should include:

- shot framing
- camera movement
- environment
- trader protagonist action
- visual metaphor
- Pump token image integration (if relevant)
- lighting + color palette
- atmospheric effects
- sound design

### Sound Design Notes (Veo with sound)

Every scene must include sound cues. Sound should evolve with acts and arc.

Example categories:

- rain on glass
- keyboard clicks
- heartbeat bass
- distant thunder
- casino crowd murmur
- glitch synth tension
- orchestral rise
- hollow room tone
- electric hum
- siren-like market tension
- morning ambience

## Veo-Specific Style Notes

- Output one continuous prompt with Scene 1..N sections.
- Keep continuity: same protagonist, evolving world, recurring motifs.
- Keep token image anchors "in-world": posters/billboards/holograms/shrines/reflections.
- Keep dialogue minimal; prefer director-style sequence descriptions.
- No stat overlays. No scoreboard numbers. No "PnL captions." No literal charts as the only subject.
